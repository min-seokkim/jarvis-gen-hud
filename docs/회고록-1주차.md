- 프로젝트 기획 확정: 주제(자비스형 엔지니어 도우미), 핵심 기능(generative HUD), 기술 스택 + 선택 이유.
- **낯선 기능 실험(프로토타이핑):** Hermes Agent를 두뇌로 세팅하고 실제로 대화가 도는 것까지 검증. 프로바이더·배포·연결 방식 의사결정.
- 배포 구조 설계(워크스테이션 서버 + 노트북 클라이언트, 포트포워딩+DDNS+Caddy) 및 `deploy/` 파일 작성.
- 2주 로드맵(M0–M6)을 라이브 아티팩트로 작성, 하이브리드 구조 타당성 확인.

## Keep (잘한 것, 유지)
- **"기능 먼저 개발하지 말고 실험·결정" 이라는 1주차 취지에 충실했다.** 서둘러 코딩하지 않고 가장 큰 미지수(에이전트 두뇌, 음성 지연, 배포 경로)를 먼저 검증했다.
- **교체 가능 구조로 설계한 게 실제로 효과를 봤다.** 두뇌를 모델 무관하게 추상화해 둔 덕에, Claude 경로가 막혔을 때 OpenAI로 바꿔 바로 진행할 수 있었다.
- **막힐 때마다 근거를 찾아 결정**했고(공식 문서·이슈 트래커 확인), 결정 이유를 문서로 남겼다.

## Problem (문제, 아팠던 것)
- **환경 세팅에 시간이 과도하게 들었다.** WSL 이미지를 드라이브 이동하는 과정에서 vhdx가 손상돼, "bad marshal" → 결국 디스크 I/O 에러까지 한 손상이 여러 얼굴로 계속 터졌다.
- **곁가지(인프라)에 끌려다녔다.** sudo 권한, Docker 그룹/통합, PATH, OAuth 사용량 게이트 등 평가 대상(프론트엔드)이 아닌 곳에서 시간을 많이 썼다.
- Claude Max 구독을 에이전트에 OAuth로 붙이려다 막혔다("out of extra usage") — 서드파티 도구의 알려진 제약.

## Try (다음 주에 시도할 것)
- **WSL 디스크는 절대 수동/임의로 옮기지 않는다.** 옮길 땐 `wsl --manage --move` 또는 설치 시 `--location`만. 백업도 명령으로.
- **인프라는 "되는 최소 경로"로 빠르게 통과**하고 본 작업(generative HUD)으로 넘어간다. 개발=네이티브, 배포용 Docker는 막바지에.
- **두뇌는 API 키(OpenAI) 기준**으로 진행하고, 구독 OAuth 같은 불확실 경로는 폴백으로만.
- 2주차는 M0 나머지(음성 스모크 테스트·react-live 샌드박스) → M1 앱 셸/스트리밍 대화로. 평가 핵심 M3·M4를 위한 토대를 만든다.

## 학습 목표 대비
- ✅ 프로젝트 기획 완성 / ✅ 워크플로우 흐름 초안 설계(별도 문서) — 1주차 목표 달성.
- 배운 것: **에이전트는 속도를 주지만, 환경·인프라의 함정은 결국 내가 판단·복구해야 한다.** 그래서 "내가 검증할 체크포인트"를 workflow에 명시했다.
