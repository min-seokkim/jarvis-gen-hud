# 자비스형 엔지니어 도우미 — 프로젝트 핸드오프 문서

> 이 문서는 브레인스토밍 단계에서 합의된 프로젝트의 성격·목적·설계 논리·확정 스택을 정리한 것이다. Cowork에서 구현을 시작할 때 이 문서를 맥락의 출발점으로 삼는다.
>
> ⚠️ **갱신 메모:** 이 문서는 **초기 브레인스토밍 기록**이다. 일부 '확정 스택'은 이후 수정됨 — 특히 **음성=로컬 Qwen / 출력양식 라우팅은 폐기**되고 **단일 Hermes 브레인 + 역할 하이브리드**(빠른 메인 + 강한 delegation)로 확정됐다(`docs/decisions/0001-framework-hermes.md`). generative HUD·자비스 비전 등 본질은 유효. 현행 진실 = `docs/기획서.md` · `AGENTS.md` · `docs/decisions/`.

---

## 1. 한 줄 정의

영화 아이언맨의 자비스를 모델로 한, **음성으로 대화하고 작업 맥락에 맞는 UI를 그 자리에서 생성해내는 엔지니어 통합 작업 도우미**. 동시에 프론트엔드 부트캠프의 기말 프로젝트이기도 하다.

---

## 2. 프로젝트의 성격과 목적

### 무엇을 만드는가
영화 속 자비스는 일정·이메일을 챙기는 개인 비서가 아니라, 토니 스타크의 **R&D 워크벤치 전체를 운영하는 엔지니어링 파트너**였다. 슈트를 시뮬레이션하고, 홀로그램 모델을 띄우고, 구조를 해석하고, 데이터를 뒤지고, 제조를 지휘했다. 이 프로젝트는 그 본질 — "엔지니어와 함께 일하는 도우미" — 을 지향한다.

### 진짜 알맹이: Generative HUD
자비스의 핵심은 멋진 고정 인터페이스가 아니라 **작업이 바뀌면 인터페이스가 따라 생성된다**는 점이다. 엔지니어링은 작업마다 필요한 표현이 다르기 때문에(회로면 회로도, 신호면 파형, 데이터면 차트) 고정 UI 하나로 덮을 수 없다. 그래서 "맥락을 읽고 거기 맞는 UI를 LLM이 실시간 생성"하는 generative HUD가 이 프로젝트의 독창적 핵심이자, 직접 0부터 짓는 유일한 부분이다.

### 성격: 프론트엔드 수업 프로젝트
이 프로젝트는 백엔드 프로젝트도 AI 모델링 프로젝트도 아니다. 평가 대상은 **generative HUD를 구현한 React 프론트엔드**다. 따라서 두뇌·음성·도구 합성처럼 이미 잘 만들어진 것은 적극적으로 갖다 쓰고, 역량은 프론트엔드(generative HUD)에 집중한다.

---

## 3. 여기까지 온 배경과 설계 논리

이 스택은 다음의 사고 흐름을 거쳐 수렴했다. 각 결정의 "왜"를 이해하는 것이 구현 중 판단에 중요하다.

1. **자비스를 만들 수 있는 시점이 왔다.** 음성 인식·합성, LLM, 도구 표준(MCP), 영속 메모리, 코드 실시간 생성이 모두 성숙했다. 빠져 있던 단 하나는 "에이전트가 맥락을 읽고 UI 생성을 자동 트리거하는 루프"였고, 그것이 generative HUD다.

2. **백엔드 늪을 경계한다.** 프론트엔드 수업 프로젝트이므로, 두뇌·도구·메모리를 직접 짜기 시작하면 정작 평가 대상인 프론트엔드를 못 끝낸다. 그래서 완성형 에이전트에 위임하기로 했다.

3. **도구 통합도 "합성"한다.** 솔리드웍스/KiCad/ANSYS 같은 도구 어댑터를 사람이 미리 다 만드는 게 아니라, 필요할 때 에이전트가 어댑터를 스스로 짜내게 한다. 이것은 generative HUD와 **완전히 같은 사상**이다 — 정적 카탈로그가 아니라 합성 엔진. UI도 생성, 도구도 생성.

4. **Hermes가 그 합성 능력을 이미 갖췄다.** Nous Research의 Hermes Agent는 자율 스킬 생성·영속 메모리·학습 루프를 내장한 자기개선 에이전트다. 우리가 만들려던 self-extending 능력을 이미 제공하므로 두뇌·스킬·도구 합성을 통째로 위임한다. 그 결과 프로젝트의 무게중심이 자연스럽게 generative HUD(아무도 안 한 우리만의 것)로 이동한다.

5. **음성이 핵심 인터페이스다.** 자비스는 말로 대화한다. 음성 출력은 사람이 듣는 속도에 묶이므로 raw 생성 속도보다 **첫 응답까지의 지연(TTFT)**이 결정적이다. 이 때문에 음성 채널은 네트워크 왕복이 없는 로컬 모델이 유리하고, 무거운 추론·UI 생성은 클라우드가 맡는 라우팅이 나온다.

---

## 4. 관통하는 핵심 원칙

구현 중 판단이 갈릴 때 아래 원칙으로 되돌아온다.

- **갖다 쓸 수 있는 건 갖다 쓴다.** 두뇌(Hermes), 보이스(ElevenLabs), 음성 인식(faster-whisper)은 위임. 0부터 짓는 것은 generative HUD 하나.
- **계산은 deterministic, LLM은 오케스트레이션.** 엔지니어링은 환각이 곧 사고다. 실제 계산·시뮬레이션·검토는 실제 코드/도구가 deterministic하게 수행하고, LLM은 "무엇을 실행하고 결과를 어떤 HUD로 보여줄지"만 지휘한다.
- **정적 카탈로그가 아니라 합성 엔진.** UI도 도구 어댑터도 미리 다 만들지 않고 필요할 때 생성한다.
- **출력 양식 기반 라우팅.** 음성 응답은 로컬(TTFT), 화면 생성·추론·합성은 클라우드(능력).
- **자유 생성 + 디자인 언어 제약.** UI는 자유롭게 생성하되, 디자인 토큰·컴포넌트·모션을 시스템이 강제해 "막 생성"되지 않고 일관된 자비스의 얼굴을 유지한다.

## 5. 확정 기술 스택

| 영역 | 결정 | 비고 |
|---|---|---|
| 두뇌 / 에이전트 | **Hermes Agent** | WSL2 + 도커 격리. 도구 합성·메모리·스킬 위임 |
| 로컬 모델 | **Qwen 3.5 9B** | 음성 즉답·의도 분류·라우팅. 한국어 강함, STT와 GPU 공존 여유 |
| 클라우드 모델 | **교체 가능 구조** | Hermes 멀티 프로바이더(OpenAI/Anthropic/Google). 추론·JSX 생성·어댑터 합성 담당. 호출부는 모델 무관하게 추상화 |
| 라우팅 | 음성 = 로컬 / 화면·추론 = 클라우드 | 출력 양식 기반 |
| STT | **faster-whisper (로컬)** | 한국어 + 영어 멀티링궐. 네트워크 0 |
| TTS | **ElevenLabs 자비스 보이스** | 라이브러리 보이스 ID 사용, multilingual_v2로 한국어, Flash v2.5 저지연 |
| 음성 목표 지연 | **왕복 0.5초** | STT 부분결과 → LLM → TTS 청크를 스트리밍 파이프라인 |
| 프론트엔드 | **React 반응형 웹앱** | 폰/PC 단말을 하나의 웹앱으로 커버. generative HUD가 핵심 |
| HUD 렌더링 | **제약 JSX** | 샌드박스(react-live/Sandpack)에서 렌더. 디자인 시스템 컴포넌트+토큰만 스코프 노출. 자기치유 루프(에러 → 재생성). iframe 격리 |
| 디자인 언어 | **자비스 클리셰** | 청록 홀로그램, 검은 배경, 파랑(안정)↔빨강(경고) 상태 매핑 |
| 도구 합성 | Hermes 스킬 시스템 위임 | 필요 시 어댑터를 런타임 합성 |

### 하드웨어 (개발/구동 환경)
RTX 4080 Super (16GB VRAM) · AMD 5950X · 64GB RAM. GPU에는 faster-whisper(STT)와 Qwen 3.5 9B만 상주(TTS는 클라우드라 GPU 미사용). 둘 합쳐 약 10~12GB로 컨텍스트 여유 확보.

## 6. 아직 열린 / 구현하며 좁힐 것

- 클라우드 추론 모델의 구체 선택 (작업별 최적이 다를 수 있으므로 교체 구조 위에서 실험)
- HUD 디자인 토큰·프리미티브 어휘·모션 언어의 구체 정의 (자비스 클리셰 방향은 확정, 세부는 구현하며)
- 자기치유 루프의 재시도 횟수·실패 시 폴백 동작
- 음성 활동 감지(VAD) 및 발화 종료 감지 튜닝

---

## 7. 첫 단계 (de-risk)

1. **Hermes 설치 후 게이트웨이 웹소켓으로 메시지 한 번 주고받기.** 이게 되면 백엔드 경로가 검증된다.
2. 반응형 React 웹앱에서 Hermes 웹소켓에 연결 — 토큰 스트리밍 대화부터.
3. 그 위에 generative HUD(제약 JSX 샌드박스 + 자비스 디자인 토큰)를 한 종류 얹어 본다.
4. 음성 파이프라인(faster-whisper → Qwen 3.5 9B → ElevenLabs) 배선. 참고로 STT→LLM→TTS를 묶은 오픈소스 자비스 비서(AlexandreSajus/JARVIS)가 파이프라인 레퍼런스로 유용.

---

## 8. 데모 비전

음성 + HUD가 **동시에** 일어나야 한다. 음성만 답하면 일반 음성비서로 보이고, 우리 알맹이인 generative HUD가 데모에 안 나온다.

- **진입(보편 후크):** "프로젝트 빌드 상태 보여줘" → 자비스가 음성으로 짧게 답하면서 동시에 화면에 빌드 진행 HUD가 생성됨(진행 막대, 단계, 빨강으로 뜨는 실패 단계). 청중이 도메인을 몰라도 즉시 이해.
- **마법(핵심 증명):** "본 적 없는 작업도 그 자리에서 HUD를 발명한다"를 한 번 보여준다. 미리 만들지 않은 작업을 던졌을 때 인터페이스가 새로 태어나는 순간이 자비스의 진짜 영역이다.
- 청중이 모르는 도메인 특화 도구(KiCad 등)는 첫 화면용이 아니라, 합성 능력을 증명할 때 곁들이는 정도로.

발표/구현 우선순위: 반응형 웹앱 + Hermes 연결 + generative HUD 한 바퀴가 도는 최소 루프를 먼저 확보하고, 음성·도구 합성·하이브리드 라우팅은 그 위에 얹는다.
